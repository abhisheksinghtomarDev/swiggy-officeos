Architecture

Event-driven backend for workplace commerce workflows.

The planned stack uses .NET 9 Web API, Angular, PostgreSQL, Redis, RabbitMQ, and SignalR. Slack and Teams events trigger structured workflows that can call Swiggy Food and Instamart MCP tools after the right checks.

Agent workflow layer Interprets requests and prepares structured order or pantry plans.
Approval service Requires admin confirmation for sensitive or budgeted workplace actions.
Audit and monitoring Tracks workflow history, approvals, errors, and integration events.

Roadmap

Built in small, reviewable steps.

Phase 1 Slack MVP

Deployment-night ordering, pantry refill suggestions, admin approval, and audit log.

Phase 2 Pilot readiness

Teams support, recurring templates, budget policies, and real-time dashboard updates.

Phase 3 Workplace scale

Multi-office controls, usage reporting, enterprise policies, and stronger monitoring.
